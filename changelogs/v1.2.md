# v1.2 — Actual Spending, Fullscreen Charts & Help System

**Released:** 2026-03-31

A major feature release adding actual spending tracking, fullscreen chart mode with interactive zoom, a built-in help system, and comprehensive documentation.

---

## Highlights

- **Actual spending tracker** — Record real transactions and link them to forecast expenses. The projection engine uses actual amounts where they exist, forecasts where they don't.
- **Fullscreen charts with zoom** — Expand any chart to fill the screen. Click-to-select zoom, range sliders, and chart-specific display toggles (Area/Line, Grouped/Stacked, Donut/Full Pie).
- **Built-in help system** — 11-topic guide accessible from the navbar and contextual ? buttons on each panel.
- **Transfer exclusion from spending charts** — Transfers between your accounts are no longer counted as "spending" in analysis charts.
- **21 documentation guides** — 12 for users, 9 for developers, with icons, screenshots, and cross-links.

---

## All Changes

### Actual Spending
- New `ActualSpend` database model with optional `forecastExpenseId` link
- Server route: `GET/POST/PUT/DELETE /api/accounts/:accountId/actuals`
- Category auto-inheritance from linked forecast expense on creation
- Projection engine: actuals fire before forecasts in `applyDay()`. Linked forecast expenses are skipped via `coveredForecastIds` set. Events tagged with `isActual: true`.
- Client: `ActualSpendList` collapsible panel (amber themed) with inline add/edit form
- Forecast expense dropdown in add form shows active expenses with interval and amount
- Visual distinction: amber dots and "actual" badges in chart tooltips and ledger rows

### Fullscreen Charts
- New `ChartFullscreen` component — fixed fullscreen overlay with toolbar and chart area
- **Zoom Select**: click two points on the chart to zoom into that date range
- **Range sliders**: two HTML range inputs at bottom for dragging start/end dates
- **Reset Zoom**: button appears when zoomed, Escape key cascades (cancel select > reset zoom > close)
- **Chart-specific controls**: Area/Line (Projection), Grouped/Stacked (Income vs Expenses), Donut/Full Pie (Spending), Stacked/Individual (Trends)
- Expand button appears on hover over any chart card
- Dynamic chart height based on viewport with resize listener
- SVG gradient IDs prefixed with `fs-` in fullscreen to avoid DOM ID collisions
- `ChartFullscreenOptions` type added to `types.ts`
- All 5 chart components accept optional `options` prop (backward compatible)

### Category System Enhancements
- Full CRUD for categories: create, rename (migrates all expenses), delete (clears from expenses), update color/description
- `description` field added to `CategoryColor` model
- Auto-discovery: `GET /api/categories` scans expenses and creates missing category entries
- Category picker modal in expense form: search, color dots, descriptions, create-new section
- Replaced dropdown combo-box with full modal dialog for better UX

### Transfer Improvements
- `isTransfer` flag added to `ProjectionEvent` interface
- Spending analysis charts (Pie, Bar, Cash Flow, Trends) exclude transfer events
- Projection chart and Ledger still include transfers (they affect balance)
- Transfer expenses grouped under "Transfer To [Account]" in expense panel

### Spreadsheet Enhancements
- New **Actual Spending** sheet in export/import (7 sheets total)
- **Category description** column added to Category Colors sheet
- **Is Transfer** and **Transfer To Account** columns on Expenses sheet
- Balance date properly preserved on import (was always using current date)
- Import validates `forecastExpenseId` — orphaned links import as unlinked with warning
- Import auto-creates missing transfer target accounts
- Instructions sheet updated with Actual Spending section
- Import results toast shows actual spend counts

### Help System
- `HelpPanel` component with sidebar navigation and 11 topics
- Topics: Getting Started, Accounts, Forecast Income, Forecast Expenses, Transfers, Actual Spending, Charts, Ledger, Categories, Spreadsheet Exchange, Tips & Workflows
- Contextual ? help buttons on navbar, chart area, and each panel header
- Cross-linked topics for easy navigation

### UI Polish
- Collapsible panels show current-month totals using interval-aware calculation (`monthlyTotal.ts`)
- "Forecast Income" and "Forecast Expenses" renamed from "Income" and "Expenses"
- `formatDate` fixed to handle full ISO strings (resolved "ends Invalid Date" bug)
- Nested button HTML errors fixed in collapsible panel headers

### Documentation
- 12 user guides with section icons, screenshots, callout boxes, welcoming intros, and See Also links
- 10 developer guides with section icons, code block language tags, Pattern/Warning/Tip callouts, and Related links
- New `docs/user/actual-spending.md` guide
- GitHub Pages landing site with privacy-first messaging, 8 screenshots, and "Built Different" section
- Docker Desktop setup instructions for macOS and Windows 11
- `CONTRIBUTING.md` with guidelines, key patterns, and privacy principles
- CLAUDE.md expanded with state management, transfer naming, and fullscreen patterns
- README expanded with full screenshot gallery

### Bug Fixes
- Balance date ignored on spreadsheet import — now preserved from exported date
- `formatDate` crashed on ISO strings containing "T" — now strips time portion
- Nested `<button>` inside `<button>` in collapsible headers — moved help buttons to siblings
- ChartFullscreen crashed on empty projections array — added bounds checks
- SVG gradient IDs clashed between normal and fullscreen views — added `fs-` prefix
- Spreadsheet import `forecastExpenseId` foreign key violation on DB reset — now validates and falls back
- Spreadsheet import drops transfer status when target account doesn't exist — now auto-creates the account

## Migration Notes

- Database schema change: `ActualSpend` model added. Run `make restart` to apply via `prisma db push`.
- No breaking API changes — all new endpoints are additive.
- `ChartType` moved from `App.tsx` to `types.ts` — import path change for any code referencing it directly.
